# AirMouse — 9축 IMU 기반 블루투스 에어마우스

> 손의 기울기와 움직임만으로 PC 마우스를 무선 제어하는 Arduino 프로젝트입니다.

## 동작 원리 요약

```
[LSM9DS1 9축 IMU]
  가속도계 + 자이로스코프 + 지자계
        │
  [칼만 필터]  ←─ Roll / Pitch 추정
        │
  [보완 필터]  ←─ Yaw 추정 (자이로 적분 + 지자계 융합)
        │
  [ZUPT 속도 보정]  ←─ 정지 시 드리프트 제거
        │
  [데드존 + 비대칭 스케일링]
        │
  [Bluetooth HID Mouse]  ──→  PC 커서 이동
```

|------|------|
| 보드 | Arduino Nano 33 BLE |
| IMU | LSM9DS1 (가속도 + 자이로 + 지자계, 내장) |

## 통신 프로토콜

| 프로토콜 | 용도 | 적용 방식 |
|----------|------|-----------|
| **UART** | 디버그 로그 출력 | `Serial.begin(115200)` 으로 초기화, `Serial.print()` 로 가속도·속도·커서 이동값 실시간 출력 |
| **I2C** | MCU ↔ LSM9DS1 센서 통신 | `Arduino_LSM9DS1` 라이브러리를 통한 인터페이스. Arduino Nano 33 BLE에서 LSM9DS1은 내부 I2C 버스로 연결되며, `IMU.readAcceleration()` 등 API 호출 시 I2C 트랜잭션이 발생 |
| **Bluetooth LE** | MCU → PC 무선 HID 전송 | BLE HID Mouse 프로파일 사용. `mouse.move(dx, dy)` 호출 시 BLE GATT를 통해 커서 이동 패킷 전송 |

## 핵심 알고리즘

### 1. 칼만 필터 (Kalman Filter)

자이로스코프와 가속도계를 융합해 Roll / Pitch 각도를 추정합니다.

| 파라미터 | 값 | 역할 |
|----------|----|------|
| `Q_angle` | 0.001 | 각도 프로세스 노이즈 (작을수록 자이로 신뢰) |
| `Q_bias` | 0.003 | 바이어스 프로세스 노이즈 |
| `R_measure` | 0.03 | 측정 노이즈 (가속도계 신뢰도) |

**예측 단계** — 자이로 적분으로 각도 예측, 오차 공분산 행렬 업데이트  
**보정 단계** — 가속도계 측정값으로 칼만 게인 계산 후 각도 및 바이어스 보정

### 2. Yaw 추정 (보완 필터)

```
yaw = 0.85 × (자이로 적분) + 0.15 × (기울기 보정 지자계)
```

자이로는 단기 정확성, 지자계는 장기 기준점을 제공합니다.

### 3. 가속도 편향 보정 (Bias Calibration)

시작 후 약 1초(100 샘플) 동안 정지 상태의 평균 가속도를 측정해 편향을 제거합니다.

### 4. ZUPT (Zero-velocity Update)

```cpp
if (|accX| < 0.1g && |accY| < 0.15g) → vx = vy = 0
```

정지 상태를 감지하면 속도를 강제 초기화해 누적 드리프트를 방지합니다.

### 5. 마우스 이동량 계산

회전 기반 이동과 가속도 적분 기반 선형 이동을 합산합니다.

```
dx = −ΔYaw × 10  +  vx × dt × 150
dy = −ΔPitch × 7  +  vy × dt × 150
```

## 중간 발표 이후 개선 사항

중간 발표 코드 리뷰 후 다음 문제를 직접 발견하고 수정했습니다.

| 구분 | 기존 문제 | 수정 내용 |
|------|-----------|-----------|
| **지자계 필터 버그** | `prevmx/y/z`가 루프 내부에서 매번 0으로 초기화되어 LPF가 사실상 `mx × 0.15`로 동작 | 전역 변수로 이동, 프레임 간 상태 유지 |
| **무의미한 코드** | `mouse.move()` 이후 스케일 되돌리기 블록 존재 (dx/dy는 다음 프레임에서 새로 계산되므로 아무 효과 없음) | 블록 제거 |
| **사용되지 않는 변수** | `accMag` 계산 후 미사용 | 제거 |
| **매직 넘버** | 스케일 값, 임계값 등이 코드 전체에 흩어진 리터럴 숫자 | 상수(`ALPHA`, `YAW_X_SCALE` 등)로 분리 |
| **불필요한 캐스트** | `(float)(dx_rot + dx_lin)` — 피연산자가 이미 float | 제거 |

## 알려진 한계 및 향후 개선 가능성

- **이중 필터링** — 칼만 필터 출력에 EMA를 추가로 적용해 응답 지연이 발생할 수 있습니다. 칼만 파라미터 튜닝만으로 대체 가능합니다.
- **비대칭 스케일링** — 방향별 감도를 수동으로 다르게 설정했습니다. AHRS 기반 좌표 변환으로 더 일관된 움직임을 구현할 수 있습니다.
- **Roll 미활용** — 현재 Roll 변화량은 마우스 이동에 반영되지 않습니다. 스크롤 등 다른 기능에 활용할 수 있습니다.
- **X_OFFSET 상수** — 하드웨어 특성에 따라 조정이 필요합니다.

→ Roll/Pitch는 **Kalman**, Yaw는 **지자계 융합형 Complementary** 필터를 선택적으로 결합하여  
평균 오차 **3.4도 이내 (Raw 대비 99.8% 개선)** 달성

## 사용 방법

1. Arduino IDE에 아래 라이브러리를 설치합니다.
   - `Arduino_LSM9DS1`
   - `ArduinoBLE` (HIDMouse 포함)

2. `librariy_KalmanSimple` 폴더를 Arduino 라이브러리 디렉토리에 추가합니다.

3. `AirMouse_Kalman_Final.ino`를 Arduino Nano 33 BLE에 업로드합니다.

4. PC의 블루투스 설정에서 **"1조 Mouse"** 로 페어링합니다.

5. 보드를 손에 쥐고 기울이면 커서가 이동합니다.
